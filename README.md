# yasno-copywriter

База знаний и skill для написания текстов Ясно в одном месте. Репозиторий подходит для трех сценариев:

- `Claude Code` читает `CLAUDE.md` и автоматически работает по правилам бренда.
- `Codex` использует skill `skills/yasno-copywriter/` с `SKILL.md` и `references/`.
- `ChatGPT` можно быстро накормить нужными `.md`-файлами вручную, если нужен разовый текст без локальной настройки.

Репозиторий хранит не только Tone of Voice, но и сам рабочий алгоритм: как определить канал, задачу текста, какие правила загрузить, как написать черновик и как проверить его перед финальной отправкой.

## Алгоритм работы

Это главный принцип репозитория. Любой хороший текст проходит один и тот же путь:

### 1. Определить канал

- Instagram
- Telegram
- VK
- Лонгрид
- Email
- Лендинг
- Бот
- Другой текст

### 2. Определить задачу

- `promo` — мягко продать или подвести к действию
- `educational` — объяснить сложную мысль
- `announcement` — анонсировать событие, запуск, обновление
- `engagement` — вызвать отклик, обсуждение, сохранение
- `sales` — ближе к конверсии, но без агрессии
- `support` — ответы в саппорте, на негатив, в сложных ситуациях
- `repurposing` — адаптировать готовый текст под другой формат

### 3. Загрузить базовые правила

Всегда читаем:

- `editorial-policy.md`
- `tov.md`
- `editing-checklist.md`
- `ethics-and-restrictions.md`
- `terminology.md`

### 4. Дозагрузить контекст канала

Потом читаем файл из `references/channels/`, который соответствует площадке.

### 5. Дозагрузить контекст задачи

Потом читаем файл из `references/tasks/`, который соответствует цели текста.

### 6. Найти релевантные примеры

Если есть подходящие примеры, смотрим в `references/examples/`.

### 7. Сначала придумать угол подачи

Перед самим текстом стоит определить:

- для кого пишем
- что человек уже знает о теме
- какое чувство или мысль должны остаться после текста
- какое действие нужно на выходе
- какой угол подачи лучше работает: поддерживающий, объясняющий, провокативный, продающий

### 8. Написать текст

Обычно полезно сразу делать:

- 2-3 угла подачи
- основной вариант
- альтернативный заголовок или первый экран
- 2-3 CTA, если они нужны

### 9. Прогнать текст через self-check

Перед финалом проверяем:

- нет ли штампов, канцелярита, нравоучений
- нет ли обещаний и чрезмерных обобщений
- нет ли языка стыда, давления, манипуляции
- понятен ли текст человеку, который не читал Ясно раньше
- начинается ли он с крючка
- ощущается ли как маленькая терапевтическая сессия, а не как бездушный маркетинг

### 10. Отдать результат в нужном формате

В зависимости от задачи финал может включать:

- заголовок
- подзаголовок
- preview text
- основной текст
- CTA
- сокращенную версию
- адаптацию под другой канал

## Как использовать в Claude Code

Если вы хотите, чтобы Claude Code автоматически учитывал весь copy system, положите репозиторий рядом с проектом и в корне проекта создайте:

```markdown
@../yasno-copywriter/CLAUDE.md
```

Путь зависит от структуры папок.

## Как использовать в Codex

Подключайте skill из папки:

```text
skills/yasno-copywriter/
```

Главный вход:

```text
skills/yasno-copywriter/SKILL.md
```

## Как использовать в ChatGPT

Если нужен быстрый one-off сценарий, достаточно загрузить или вставить в чат:

- `editorial-policy.md`
- `tov.md`
- `editing-checklist.md`
- файл канала
- файл задачи

Этого уже хватает, чтобы получить внятный черновик без generic marketing copy.

## Production Workflow

Ниже — рабочий способ использовать репозиторий не как архив, а как production system.

### Шаг 1. Дайте короткий бриф

Минимальный бриф должен содержать:

- канал
- задачу
- тему
- аудиторию
- желаемое действие
- ограничения по формату

Хороший короткий бриф выглядит так:

```text
Канал: Telegram
Задача: educational + promo
Тема: почему сложно злиться
Аудитория: взрослые, которые привыкли подавлять злость
Действие: мягко подвести к терапии
Ограничения: без кликбейта, нужен один основной пост и 3 варианта первой фразы
```

### Шаг 2. Запустите алгоритм

Модель должна пройти этот порядок:

1. core rules
2. channel file
3. task file
4. relevant examples
5. choose angle
6. draft
7. self-check
8. final output in channel format

### Шаг 3. Не просите “просто написать текст”

Лучше просить один из production-режимов:

- `сгенерируй с нуля`
- `дай 3 угла подачи`
- `перепиши в стиле Ясно`
- `адаптируй из Telegram в Instagram`
- `сделай более коротко / теплее / яснее`
- `проверь на соответствие редполитике`

### Шаг 4. Требуйте правильный output format

Для каждого канала полезно заранее фиксировать формат ответа.

`Telegram`

- 3 opening lines
- main post
- optional CTA
- optional comments prompt

`Instagram`

- 3 first-slide options
- slide-by-slide structure
- caption
- optional story bridge

`VK`

- 3 hook options
- main post or clip-support text
- poll/comment prompt if relevant

`Email`

- 3 subject lines
- preview text
- body
- CTA options

`Landing`

- hero options
- subhead
- section structure
- proof points
- CTA blocks

`Bot`

- message-by-message flow
- buttons
- fallback wording if needed

`Longread`

- thesis
- outline
- full text or draft sections
- soft service bridge

### Шаг 5. Проверяйте не “красиво ли”, а “работает ли”

Хороший текст для Ясно:

- звучит как Ясно, а не как generic marketing
- уважает читателя
- не обещает чудес
- не давит
- не стыдит
- дает ясную мысль, а не просто настроение
- учитывает реальную механику канала

## Production Prompts

Ниже — готовые шаблоны, которые можно использовать почти без изменений.

### Universal Prompt

```text
Используй yasno-copywriter как основной copy system.

Сначала:
1. Определи канал.
2. Определи задачу текста.
3. Подгрузи базовые правила бренда.
4. Подгрузи файл канала.
5. Подгрузи файл задачи.
6. Если есть релевантные примеры, используй их.

Потом:
1. Коротко сформулируй 2-3 возможных угла подачи.
2. Выбери лучший.
3. Напиши текст в формате нужного канала.
4. Проведи self-check по редполитике, ToV, этике и editing checklist.
5. Убери штампы, давление, кликбейт, поп-психологию и манипуляции.

Бриф:
[вставьте сюда бриф]
```

### Telegram Prompt

```text
Напиши Telegram-пост в стиле Ясно.

Нужно:
- 3 варианта первой фразы
- 1 основной пост
- мягкий CTA только если он правда нужен

Проверь, чтобы пост:
- не был кликбейтным
- не звучал как коучинг
- давал новый психологический взгляд
- не был перегружен теорией

Тема:
[тема]

Аудитория:
[аудитория]

Задача:
[educational / promo / engagement / announcement]
```

### Instagram Prompt

```text
Сделай Instagram-карусель в стиле Ясно.

Верни:
- 3 варианта первого слайда
- структуру по слайдам
- подпись к посту

Учитывай:
- главная мысль должна быть в карточках
- подпись не должна дублировать карусель целиком
- в карточках нужен воздух
- один слайд = одна мысль

Тема:
[тема]

Задача:
[задача]
```

### Email Prompt

```text
Напиши письмо в стиле Ясно.

Верни:
- 3 темы письма
- preview text
- основной текст
- 2-3 CTA

Проверь, чтобы письмо:
- держалось на одной главной идее
- было полезным само по себе
- мягко подводило к действию
- не звучало как рассылка ради конверсии

Бриф:
[бриф]
```

### Repurposing Prompt

```text
Адаптируй этот текст в стиль Ясно для нового канала.

Сначала коротко объясни:
- что нужно сохранить из исходника
- что нужно переписать из-за канала
- какой угол подачи лучше выбрать

Потом верни финальную версию.

Исходный канал:
[канал]

Новый канал:
[канал]

Текст:
[исходный текст]
```
