# yasno-copywriter

База знаний и skill для написания текстов Ясно в одном месте. Репозиторий подходит для трех сценариев:

- `Claude Code` читает `CLAUDE.md` и автоматически работает по правилам бренда.
- `Codex` использует skill `skills/yasno-copywriter/` с `SKILL.md` и `references/`.
- `ChatGPT` можно быстро накормить нужными `.md`-файлами вручную, если нужен разовый текст без локальной настройки.

Репозиторий хранит не только Tone of Voice, но и сам рабочий алгоритм: как определить канал, задачу текста, какие правила загрузить, как написать черновик и как проверить его перед финальной отправкой.

## Алгоритм работы

Это главный принцип репозитория. Любой хороший текст проходит один и тот же путь:

### 1. Определить канал

- Instagram
- Telegram
- VK
- Лонгрид
- Email
- Лендинг
- Бот
- Другой текст

### 2. Определить задачу

- `promo` — мягко продать или подвести к действию
- `educational` — объяснить сложную мысль
- `announcement` — анонсировать событие, запуск, обновление
- `engagement` — вызвать отклик, обсуждение, сохранение
- `sales` — ближе к конверсии, но без агрессии
- `support` — ответы в саппорте, на негатив, в сложных ситуациях
- `repurposing` — адаптировать готовый текст под другой формат

### 3. Загрузить базовые правила

Всегда читаем:

- `editorial-policy.md`
- `tov.md`
- `editing-checklist.md`
- `ethics-and-restrictions.md`
- `terminology.md`

### 4. Дозагрузить контекст канала

Потом читаем файл из `references/channels/`, который соответствует площадке.

### 5. Дозагрузить контекст задачи

Потом читаем файл из `references/tasks/`, который соответствует цели текста.

### 6. Найти релевантные примеры

Если есть подходящие примеры, смотрим в `references/examples/`.

### 7. Сначала придумать угол подачи

Перед самим текстом стоит определить:

- для кого пишем
- что человек уже знает о теме
- какое чувство или мысль должны остаться после текста
- какое действие нужно на выходе
- какой угол подачи лучше работает: поддерживающий, объясняющий, провокативный, продающий

### 8. Написать текст

Обычно полезно сразу делать:

- 2-3 угла подачи
- основной вариант
- альтернативный заголовок или первый экран
- 2-3 CTA, если они нужны

### 9. Прогнать текст через self-check

Перед финалом проверяем:

- нет ли штампов, канцелярита, нравоучений
- нет ли обещаний и чрезмерных обобщений
- нет ли языка стыда, давления, манипуляции
- понятен ли текст человеку, который не читал Ясно раньше
- начинается ли он с крючка
- ощущается ли как маленькая терапевтическая сессия, а не как бездушный маркетинг

### 10. Отдать результат в нужном формате

В зависимости от задачи финал может включать:

- заголовок
- подзаголовок
- preview text
- основной текст
- CTA
- сокращенную версию
- адаптацию под другой канал

## Как использовать в Claude Code

Если вы хотите, чтобы Claude Code автоматически учитывал весь copy system, положите репозиторий рядом с проектом и в корне проекта создайте:

```markdown
@../yasno-copywriter/CLAUDE.md
```

Путь зависит от структуры папок.

## Как использовать в Codex

Подключайте skill из папки:

```text
skills/yasno-copywriter/
```

Главный вход:

```text
skills/yasno-copywriter/SKILL.md
```

## Как использовать в ChatGPT

Если нужен быстрый one-off сценарий, достаточно загрузить или вставить в чат:

- `editorial-policy.md`
- `tov.md`
- `editing-checklist.md`
- файл канала
- файл задачи

Этого уже хватает, чтобы получить внятный черновик без generic marketing copy.

## Ближайший план развития

Сейчас в репозитории есть ядро правил и каркас по каналам и задачам.

Дальше стоит добавить:

- размеченные примеры по каждому каналу
- анти-примеры

- тестовый набор задач для проверки качества skill-а
